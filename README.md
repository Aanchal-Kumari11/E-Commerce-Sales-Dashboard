# E-Commerce Sales Dashboard

## Project Overview
This project analyzes e-commerce sales data using SQL, PostgreSQL, and Power BI to identify revenue trends, product performance, profit insights, and refund patterns. The dashboard provides interactive visualizations for business decision-making and performance tracking.

---

## Tools & Technologies Used
- PostgreSQL
- Power BI
- Excel / CSV

---

## Project Workflow
1. Collected and explored e-commerce CSV datasets.
2. Imported datasets into PostgreSQL.
3. Performed SQL analysis using joins, aggregations, filtering, and KPI calculations.
4. Connected PostgreSQL database with Power BI.
5. Created interactive dashboard visualizations and KPIs.

---

## Key KPIs
- Total Revenue
- Total Orders
- Total Profit
- Average Order Value
- Total Refund Count

---

## Dashboard Features
- Monthly Revenue Trend Analysis
- Product-wise Revenue Comparison
- Refund Analysis Chart
- Interactive Year and Product Slicers
- Product Performance Matrix

---

## SQL Analysis Performed
The project includes SQL queries for:
- Revenue analysis
- Profit calculation
- Monthly sales trends
- Product performance analysis
- Refund analysis
- Aggregations and joins

---

## Key Insights
- Revenue showed continuous growth during 2014.
- The Original Mr. Fuzzy had the highest refund count.
- Top-performing products contributed significantly to overall revenue.
- Refund trends helped identify products with potential quality or customer satisfaction issues.
